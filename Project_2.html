Comparison between Locally Weighted Regression and Random Forest¶

Locally Weighted Regression (Loess or Lowess):¶

Locally weighted regression, or loess, is a non-parametric procedure for fitting a regression surface to data through multivariate smoothing: The dependent variable is smoothed as a function of the independent variables in a moving fashion analogous to how a moving average is computed for a time series 1. It performs regression step by step around a point of interest using only the data that are 'local' to that point, which means the few nearest poitns 2.

Random Forest¶

Random forest is a meta estimator that fits a number of classifying decision trees on various sub-samples of the dataset and uses averaging to improve the predictive accuracy and control over-fitting 3.

Locally Weighted Regression¶

First thing to perform is the locally weighted regression or loess. For this regression, we need Kernels–bump function. We could experiment three types of bump functions and see which is best.

In [6]:
# Tricubic Kernel
def tricubic(x):
    return np.where(np.abs(x)>1,0,70/81*(1-np.abs(x)**3)**3)

# Epanechnikov Kernel - is perhaps one of the best choices
def Epanechnikov(x):
    return np.where(np.abs(x)>1,0,3/4*(1-np.abs(x)**2))

# Quartic Kernel
def Quartic(x):
    return np.where(np.abs(x)>1,0,15/16*(1-np.abs(x)**2)**2)

Here, the function for loess regression is defined. Tau is the bandwidth that defines the area of which neighboring data should be considered when calculating the weight.

In [7]:
def lowess_reg(x, y, xnew, kern, tau):
    # tau is called bandwidth K((x-x[i])/(2*tau))
    # IMPORTANT: we expect x to the sorted increasingly
    n = len(x)
    yest = np.zeros(n)

    #Initializing all weights from the bell shape kernel function    
    w = np.array([kern((x - x[i])/(2*tau)) for i in range(n)])     
    
    #Looping through all x-points
    for i in range(n):
        weights = w[:, i]
        b = np.array([np.sum(weights * y), np.sum(weights * y * x)])
        A = np.array([[np.sum(weights), np.sum(weights * x)],
                    [np.sum(weights * x), np.sum(weights * x * x)]])
        #theta = linalg.solve(A, b) # A*theta = b
        theta, res, rnk, s = linalg.lstsq(A, b)
        yest[i] = theta[0] + theta[1] * x[i]
    # Linear interpolation for x
    f = interp1d(x, yest,fill_value='extrapolate')
    # Get the prediction
    return f(xnew)

Result¶

When comparing the mean squared errors of all of the models we have calculated, there's no big difference. That's because they are all trained to provide a result with the smallest mean sqared errors and they all succeeded to do so. But the goal of running loess is producing a smooth prediction line. While the result of the random forest is better than when the tau of the loess was low (0.05), it is not better than if the tau was big (0.1). Therefore, while we could not necessarily say that loess will always be better than random forest, through adjustments, we can make it have a smoother prediction line than random forest.
